# BookRAG

A production-grade Retrieval-Augmented Generation (RAG) system that answers questions about books and documents. Built with LangChain, FAISS, Cohere, Groq, and FastAPI.

---

## Architecture

The system is split into two independent pipelines:

```
┌─────────────────────────────────────────────────────────────────┐
│                      INGESTION PIPELINE                         │
│                                                                 │
│  Documents  →  Chunking  →  Embeddings  →  FAISS Index (disk)  │
│  (PDF/TXT/CSV)  (1000c)    (MiniLM-L6)    vectorstore/          │
└─────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────┐
│                       QUERY PIPELINE                            │
│                                                                 │
│  User Query                                                     │
│     │                                                           │
│     ▼                                                           │
│  1. Query Understanding  (clean + LLM rewrite)                  │
│     │                                                           │
│     ▼                                                           │
│  2. Retriever            (FAISS top-10, L2 → similarity score)  │
│     │                                                           │
│     ▼                                                           │
│  3. Reranker             (Cohere cross-encoder, top-5)          │
│     │                                                           │
│     ▼                                                           │
│  4. Context Builder      (dedup, token budget, format)          │
│     │                                                           │
│     ▼                                                           │
│  5. Prompt Builder       (system prompt + context injection)    │
│     │                                                           │
│     ▼                                                           │
│  6. LLM Service          (Groq / llama-3.3-70b)                 │
│     │                                                           │
│     ▼                                                           │
│  Answer  +  Observability (LangSmith traces, latency, tokens)   │
└─────────────────────────────────────────────────────────────────┘
```

## Configuration

All settings live in `config/config.py`. Update the following before running:

```python
# Groq LLM
GROQ_API_KEY = "your-groq-api-key"

# Cohere Reranker
COHERE_API_KEY = "your-cohere-api-key"

# LangSmith Tracing (optional — set LANGSMITH_TRACING_ENABLED = False to disable)
LANGSMITH_API_KEY = "your-langsmith-api-key"
LANGSMITH_PROJECT = "BookRAG"
```

Key tunable constants:

| Constant | Default | Description |
|---|---|---|
| `CHUNK_SIZE` | 1000 | Characters per chunk |
| `CHUNK_OVERLAP` | 200 | Overlap between chunks |
| `RETRIEVER_TOP_K` | 10 | Candidate chunks fetched from FAISS |
| `RETRIEVER_SCORE_THRESHOLD` | 0.5 | Min similarity score to keep a chunk |
| `RERANKER_TOP_N` | 5 | Chunks returned after Cohere reranking |
| `CONTEXT_MAX_TOKENS` | 2000 | Token budget for the context block |
| `LLM_MODEL` | `llama-3.3-70b-versatile` | Groq model |
| `LLM_TEMPERATURE` | 0.0 | Deterministic output |

## Quickstart (local)

**1. Install dependencies**
```bash
uv sync
```

**2. Add documents**

Place your PDF, TXT, or CSV files in the `data/` directory.

**3. Run ingestion**

Builds the FAISS index and saves it to `vectorstore/faiss_index/`.
```bash
uv run python src/app/rag_ingestion/ingestion_pipeline.py
```

**4. Start the API server**
```bash
uv run python src/api/main.py
```

Server starts at `http://localhost:8000`.

**5. Ask a question**
```bash
curl -X POST http://localhost:8000/ask \
  -H "Content-Type: application/json" \
  -d '{"query": "What is the attention mechanism in transformers?"}'
```

Response:
```json
{
  "request_id": "a3f1b2c4-...",
  "query": "What is the attention mechanism in transformers?",
  "answer": "The attention mechanism allows the model to ...",
  "latency_ms": 1842.5
}
```

## API Reference

### `GET /health`

Liveness and readiness check.

```json
{
  "status": "ok",
  "pipeline_ready": true,
  "version": "1.0.0"
}
```

### `POST /ask`

Ask a question against the knowledge base.

**Request body:**
```json
{
  "query": "string (3–500 characters)"
}
```

**Response:**
```json
{
  "request_id": "string (UUID)",
  "query": "string",
  "answer": "string",
  "latency_ms": "float"
}
```

**Error response (4xx / 5xx):**
```json
{
  "request_id": "string",
  "error": "string",
  "detail": "string"
}
```

Interactive docs available at `http://localhost:8000/docs`.

## Docker Deployment

**Build the image**

The build pre-downloads the HuggingFace embedding model so the container starts fast and works offline.

```bash
docker build -t bookrag:latest .
```

**Run the container**

Mount the pre-built FAISS index as a read-only volume:

```bash
docker run --rm -p 8000:8000 \
  -v "$(pwd)/vectorstore:/app/vectorstore:ro" \
  bookrag:latest
```

**Check health**
```bash
curl http://localhost:8000/health
```

## Tech Stack

| Component | Technology |
|---|---|
| Document loading | LangChain (PyMuPDF, TextLoader, CSVLoader) |
| Chunking | LangChain RecursiveCharacterTextSplitter |
| Embeddings | `sentence-transformers/all-MiniLM-L6-v2` (384-dim, CPU) |
| Vector store | FAISS (local, L2 distance) |
| Reranker | Cohere `rerank-english-v3.0` |
| LLM | Groq `llama-3.3-70b-versatile` |
| API | FastAPI + Uvicorn |
| Observability | LangSmith |
| Package manager | uv |
| Container | Docker (multi-stage, non-root) |
